# Лабораторная работа №1
## Обзор проекта

Данная работа реализует подсистему приёма задач, построенную на принципах **Duck Typing** и **контрактного программирования**. Система позволяет собирать данные из разнородных источников, не связанных иерархией наследования, обеспечивая гибкость и расширяемость.

## Ключевые архитектурные решения

1. **Инверсия зависимостей**: `TaskProcessor` зависит от абстракции `TaskSource`, а не от конкретных реализаций источников.
2. **Структурная типизация (`typing.Protocol`)**: Контракт источника описан через протокол. Это позволяет любому классу считаться «источником», если он реализует метод `get_tasks()`, без явного наследования.
3. **Runtime-проверка**: Использование декоратора `@runtime_checkable` позволяет выполнять валидацию объектов во время выполнения программы с помощью `isinstance()`.

## Быстрый старт

### Требования

* Python 3.9+

### Установка и запуск

1. **Подготовьте окружение**:
```bash
python -m venv venv
source venv/bin/activate
```


2. **Запустите демонстрацию**:
```bash
python -m src.main
```

## Логирование

Все операции системы логируются в файл `shell.log`. Настроена поддержка кодировки `utf-8` для корректного отображения кириллицы и формат вывода, включающий имя модуля для удобной отладки.

# Лабораторная работа №1

## Ключевые концепции

- **Структурная типизация (`Protocol`)**: Источники задач реализуют поведенческий контракт.
- **Runtime-проверка (`@runtime_checkable`)**: Использование `isinstance()` для проверки соответствия объекта контракту во время выполнения.
- **Расширяемость**: `TaskProcessor` работает с любым объектом, удовлетворяющим протоколу `TaskSource`, это позволяет легко добавлять новые источники.
- **Модульность**: Проект разделен на независимые компоненты (модели, контракт, источники, процессор).

## Установка и запуск

1.  **Клонируйте репозиторий** и перейдите в корневую директорию проекта.

2.  **Создайте и активируйте виртуальное окружение**:
    ```bash
    python -m venv venv
    source venv/bin/activate  # Для Windows: venv\Scripts\activate
    ```

3.  **Установите зависимости**:
    ```bash
    pip install -r requirements.txt
    ```

4.  **Запустите демонстрацию**:
    ```bash
    python -m src.main
    ```
    Эта команда соберет задачи и выведет их в консоль. Логи работы будут сохранены в файл `shell.log`.

## Запуск тестов

Для проверки корректности работы всех компонентов системы используется `pytest`.

```bash
pytest
```
Эта команда автоматически обнаружит и выполнит все тесты из папки `tests/`.
